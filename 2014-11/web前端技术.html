web前端技术

web前端技术对我来说，总是有点难以熟练掌握和理解，之前是认为前端技术只是做前台、搭界面，而我是做后台的，做底层，将来做架构的。虽然都是“做台的”，但是还是感觉后台牛B一点。结果现在发现前端技术发展太快，而且各种框架，类库，层出不穷，导致我根本不能明白前端是什么了。

所以现在希望沉下心来，谦虚一点，仔细学习一下前端技术。
我现在发现，这么多年来互联网技术日新月异，交替更迭特别快，但是终究还是有很多是不变的。比如：TCP、HTTP等网络协议、Web页面的跳转关系、Web表单，DIV布局，CSS样式。

单纯从Web开发来说，变化最多的是交互设计，和各种辅助框架。我理解各种开源框架和开源系统都是为了帮助工程师更快的开发。当然这些框架是有一定的学习成本，如何降低各种开发工具的学习成本是后面我要面对的问题。

我尝试从外行的角度，肤浅的列举一下前端技术

HTML

HTML翻译成中文就是超文本标记语言，还是一个文本的描述文档，只不过多了一些标签，可以展现更加丰富的内容，主要是得益于浏览器的强大。网络最初为了适配各种平台，基本只是传输TEXT文档，为了进一步描述文档的外观和语义，才加入了各种标签。随后产生了HTTP协议专门用于传输HTML文档。很有意思的是，先有的HTML，后来才产生的HTTP。

整个互联网就是通过普通的HTML页面连接起来，进而形成的一个文档间的连接网络。一个互联网业务就是让用户在这些页面间跳来跳去，来满足用户需求。
HTML是描述页面结构和内容，HTML5有100多个标签，多了一些复杂的功能cavnas, webWorker, SSE, Location等等。但其最主要的能力还是在内容展示方面。

CSS

CSS其实就是为HTML做美肤的。他通过各种选择器为HTML设置属性，使网页更加好看。现在CSS3增加了很多动画功能。
我没做过CSS的开发，所以我都不知道该怎么总结这个技术。CSS给我的感觉就是用来调整高度，宽度，字体，颜色，背景色等等属性的。一直是认为这个东东很繁琐且没技术含量。但是现在有很多开源的CSS类库，使得这样的工作的变得简单了，安心写好HTML框架就可以了。

Javascript

Javascript是在HTML和CSS之后才被开发出来的。他主要是增强网页的动态性，做些简单的逻辑判断，动画等等。现在Javascript功能越来越强大了，配合着Ajax、jQuery等技术，任何Web项目都不可能没有Javascript。

以上是我对前端技术的非常肤浅的认识。虽然说现在都是移动互联网时代了，全部都是客户端+后端的模式，但是我还是想等有时间了，再学习一下Bootstrap和jQuery，算是补上自己落下的那一课了。
