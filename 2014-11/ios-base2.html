应用开发都是在某个平台之上。平台和应用都是相对的概念。比如在硬件平台上开发操作系统，操作系统就是一个应用；iPhone的平台上开发iOS，iOS就是一个应用；在iOS上开发App，App就是一个应用。既然在平台上开发应用，就要依赖平台提供的所谓系统框架。

程序入口

main函数是入口，这个大部分C应用程序是一样的

#import <UIKit/UIKit.h>  

#import "AppDelegate.h"  
int main(int argc, char * argv[])
    {
                @autoreleasepool {
                            return UIApplicationMain(argc, argv, nil,  NSStringFromClass([AppDelegate class]));  
                                } }

从UIApplicationMain函数开始就进入了UIKit Framework的框架体系中了。

UIApplication,UIWidow都是由UIApplicationMain创建的系统对象。
我们主要开发的就是ViewController、View、DataObject。UIApplication负责处理主线程，相应各种用户操作事件。

ViewController

当AppDelegate指定了rootViewController后，viewController开始进行初始化，并加载相关的View，并对View进行初始化等工作。Xcode提供的StoryBoard可以简化代码，但是个人觉得非常不好用。ViewController是iOS框架中的概念，负责关联DataModel和View。UIKit有很多标准ViewController,也构成了目前App的交互基础控件。iOS提供了基础的container controller, 我们自定义的叫content controller。

  • UINavigationController
  • UITabbarController
  • UISplitViewController
  • UITableViewController
  • UIViewController
View

ViewController初始化是的时候，需要对View进行初始化。一般都是加载各个页面的初始布局，调整好Frame，放置好Label,Button,Image等等，大多时候会用到TableView, ScorllView，显示好用户界面后，就等待用户的输入，然后加载相关的数据，并刷新View。
每个UIView的背后都有个CALayer在支持，复杂的动画可以直接操作在CALayer上
