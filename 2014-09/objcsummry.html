Objective-C语言学习总结

花费太大力气学习objective-C语言是不值得的，除了能在OSX/iOS上有成熟的类库，其他领域几乎没有什么特别的类库，有个GNUstep，看上去也是远古的产物。而且很多这个语言的高级特性在iOS 应用开发时也只是用到皮毛，这些高级特性都是为cocoa framework自身开发而准备的。

基础语法

类定义和消息传递，
@interface Person : NSObject

@property NSString* firstName;   //定义属性
- (void)sayHello;    //定义方法
+ (void)newPerson;   //静态方法
@end

objc和其他面向对象语言一样，都以定义的类和对象，为语言的主要操作对象。
objc定义一个类接口，在关键词@interface、@end 之间定义类的外部属性和方法 

#import "XYZPerson.h"
@implementation XYZPerson

- (void)sayHello {
    NSLog(@"Hello, World!");
}

+ (void)newPerson {
    NSLog(@"Hello, World!");
}

@end

类的实现文件写在.m文件中。objc中类也是对象，普通对象通过class方法，返回类对象。
oc里面关键词self约等于C++里面的this, super 是一个新的关键词，而且很方便。

创建对象一般使用如下方式： 

    XYZPerson* person = [[XYZPerson alloc]init];

给对象发消息： 

    [person sayHello];

因为objc是动态语言，所以他的所有对象变量都是指针，所有的函数都是类似C++虚函数，运行的时候才会动态绑定

类的扩展

Category扩展

对任意一个可以通过Category的方式进行扩展，在没有源代码的情况下，增加类的方法 

@interface UIImageView (scale)

    - (void)scaleNewImage:(UIImage*)newImage;

@end
协议和代理
        很早学OOP的时候，感觉为了代码复用，可以抽象，然后继承。继承是类知道自己怎么实现某个方法，而当某个方法不知道该怎么做的时候，很多时候为了实现这个方法，需要把能够实现此方法的类引入，作为成员变量，但这还是非常耦合的事情。

协议就灵活多了。协议的声明类似一个类，使用@protocol关键字。在protocol中可以声明普通方法和类方法，还可以声明属性。通过声明一个protocol,无论是使用方和实现方，都可以不清楚对方的具体细节，只需要遵循固定的协议就可以了。而且任何类都可以实现某种协议，来完成相同接口的不同操作。实现松耦合。
需要注意delegate 最好加上weak修饰符，避免strong reference cycle.

Block

Block是一段可以作为对象的代码，类似其他语言的闭包或lambda.语法上就是 ^ 加上 {} 包含的一段代码 

^{  
     NSLog(@"This is a block");
 }

Block是一个对象，可以把Block复制给以一个变量，Block类型的变量声明语法类似C语言的函数指针 

void (^simpleBlock)(void);  //声明
simpleBlock();  //调用

Block内部可以获得作用域内的变量值，但是无法修改，而且当变量变化时，也只能得到最初的值，相当于Block内部拷贝一份相关变量原始的值。如果要修改变量，必须增加__block修饰符，相当于此变量在Block内部用过指针的方式被引用。

Block 作为参数传递，可以很方便的结合Operation Queue和Dispatch Queue实现多线程。

Foundation

Objc提供了很好的基础类库，类似于stl之于C++。这套基础库可用于iOS和OSX的开发。常见的基础类型NSString, NSNumber,一些集合类型NSArray, NSDictionary都属于此类库中。
具体可参考Foundation.h文件，里面记录了所有Foundation包含的类
