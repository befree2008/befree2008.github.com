iOS网络编程

iOS提供三层网络接口

  • Foundation Layer
  • Core Foundation Layer
  • Posix Layer

对于普通应用来说，一般是通过使用HTTP协议，来完成各种网络操作，包括获取数据（Get/Post），下载，上传。Foundation Layer可以完成这些任务，不必使用太底层的API。苹果对这类基于HTTP的网络操作叫”URL Loading System”，不知道这个该怎么翻中文。

iOS/OSX网络API实际上是以NSURLRequest为中心建立的，对NSURLRequest进行操作的是NSURLSession或者NSURLConnection。这两个类既可以通过Block的方式，又可以通过delegate的方式来处理相关的网络操作结果和数据。

NSURLConnection

使用NSURLConnection delegate方式实现网络通讯，代理的方式可以实现断点续传等复杂一些功能，必须实现以下几个代理函数

  • connection:didReceiveResponse:
  • connection:didReceiveData:
  • connection:didFailWithError:
  • connectionDidFinishLoading:
NSURLSession

URLSession是iOS7以后，替代URLConnection的新类，他提供一些类的新特性，可以更加灵活的配置网络请求，同时他可以支持后台数据传输，这个可以解决很多后台大文件的上传下载的问题。

URLSesssion系列的类，包括NSURLSessionConfiguration， NSURLSessionTask。Task相关的类包含三种类型。

NSStream

除了针对HTTP协议封装的NSURL相关的类，Foundation层还提供了NSStream类这样更加raw的接口，方便我们实现各种自定义的网络数据传输，比如增加SSL，SOCKS等等。NSStream基本上是CFStream的Objective-c的封装。

CFStream

Core Foundation层提供的socket接口就是CFStream相关的类，这个可以在iOS, OSX上代码通用的。CFStream框架集成了Runloop方式处理消息，创建一个CFStream，加入到runloop中，就可以处理各种stream的event了。

ARC不能管理Core Foundation Object的生命周期，所以在Core Foundation和ARC之间，我们需要使用到 __bridge, __bridge_retained和__bridge_transfer三个转换关键字。
